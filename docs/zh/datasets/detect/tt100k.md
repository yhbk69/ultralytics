# TT100K 数据集

[Tsinghua-Tencent 100K (TT100K)](https://cg.cs.tsinghua.edu.cn/traffic-sign/) 是一个大规模交通标志基准数据集，由 100,000 张腾讯街景全景图创建。该数据集专为真实世界条件下的交通标志检测和分类而设计，为研究人员和开发者提供了构建鲁棒交通标志识别系统的全面资源。

该数据集包含 **100,000 张图像**，超过 **30,000 个交通标志实例**，涵盖 **221 个标注类别**。原始论文对每个类别应用 100 个实例的阈值进行监督训练，产生了一个常用的 **45 类**子集；然而，提供的 Ultralytics 数据集配置保留了所有 **221 个标注类别**，其中许多类别非常稀疏。这些图像捕捉了光照、天气条件、视角和距离的巨大变化，使其非常适合训练需要在多样化真实场景中可靠运行的模型。

该数据集对以下方面特别有价值：

- 自动驾驶系统
- 高级驾驶辅助系统（ADAS）
- 交通监控应用
- 城市规划和交通分析
- 真实世界条件下的计算机视觉研究

## 主要特性

TT100K 数据集提供了几个关键优势：

- **规模**：100,000 张高分辨率图像（2048×2048 像素）
- **多样性**：221 个交通标志类别，涵盖中国交通标志
- **真实世界条件**：天气、光照和视角的巨大变化
- **丰富标注**：每个标志包含类别标签、边界框和像素掩码
- **全面覆盖**：包括禁令、警告、指示和信息标志
- **训练/测试划分**：预定义的划分，用于一致的评估

## 数据集结构

TT100K 数据集分为三个子集：

1. **训练集**：
   用于训练模型检测和分类不同类型交通标志的交通场景图像主要集合。
2. **验证集**：
   在模型开发期间用于监控性能和调整超参数的子集。
3. **测试集**：
   用于评估最终模型在真实世界场景中检测和分类交通标志能力的保留图像集合。

TT100K 数据集包含 221 个交通标志类别，分为几个主要组：

**限速标志（pl*, pm*）**

1. **pl\_**：禁令限速（pl5, pl10, pl20, pl30, pl40, pl50, pl60, pl70, pl80, pl100, pl120）
2. **pm\_**：最低限速（pm5, pm10, pm20, pm30, pm40, pm50, pm55）

**禁令标志（p*, pn*, pr\_）**

1. **p1-p28**：一般禁令标志（禁止驶入、禁止停车、禁止临时停车等）
2. **pn/pne**：禁止驶入和禁止停车标志
3. **pr**：各种限制标志（pr10, pr20, pr30, pr40, pr50 等）

**警告标志（w\_）**

1. **w1-w66**：各种道路危险、状况和情况的警告标志
2. 包括人行横道、急转弯、湿滑路面、动物、施工等

**高度/宽度限制标志（ph*, pb*）**

1. **ph\_**：高度限制标志（ph2, ph2.5, ph3, ph3.5, ph4, ph4.5, ph5 等）
2. **pb\_**：宽度限制标志

**信息标志（i*, il*, io, ip）**

1. **i1-i15**：一般信息标志
2. **il\_**：限速信息（il60, il80, il100, il110）
3. **io**：其他信息标志
4. **ip**：信息牌

## 使用方法

要在 TT100K 数据集上以 640 的图像大小训练 YOLO26 模型 100 个 [epoch](https://www.ultralytics.com/glossary/epoch)，可以使用以下代码片段。数据集将在首次使用时自动下载并转换为 YOLO 格式。

!!! example "训练示例"

    === "Python"

        ```python
        from ultralytics import YOLO

        # 加载模型
        model = YOLO("yolo26n.pt")  # 加载预训练模型（推荐用于训练）

        # 训练模型 - 数据集将在首次运行时自动下载
        results = model.train(data="TT100K.yaml", epochs=100, imgsz=640)
        ```

    === "CLI"

        ```bash
        # 从预训练的 *.pt 模型开始训练
        # 数据集将在首次运行时自动下载和转换
        yolo detect train data=TT100K.yaml model=yolo26n.pt epochs=100 imgsz=640
        ```

## 常见问题

### TT100K 数据集用于什么？

Tsinghua-Tencent 100K (TT100K) 数据集专为真实世界条件下的交通标志检测和分类而设计。它主要用于：

1. 训练自动驾驶感知系统
2. 开发高级驾驶辅助系统（ADAS）
3. 研究变化条件下的鲁棒目标检测
4. 基准测试交通标志识别算法
5. 测试模型在大图像中小物体上的性能

拥有 100,000 张多样化街景图像和 221 个交通标志类别，它为真实世界交通标志检测提供了全面的测试平台。

### TT100K 中有多少个交通标志类别？

TT100K 数据集包含 **221 个不同的交通标志类别**，包括：

1. **限速**：pl5 到 pl120（禁令限速）和 pm5 到 pm55（最低限速）
2. **禁令标志**：28+ 种一般禁令类型（p1-p28）加上限制（pr\*, pn, pne）
3. **警告标志**：60+ 种警告类别（w1-w66）
4. **高度/宽度限制**：ph* 和 pb* 系列物理限制
5. **信息标志**：i1-i15, il\*, io, ip 用于引导和信息

这种全面覆盖包括了中国道路网络中的大多数交通标志。

### 与其他数据集相比，TT100K 有哪些挑战？

TT100K 提出了几个独特的挑战：

1. **尺度变化**：标志范围从非常小（远处的高速公路标志）到大（近处的城市标志）
2. **真实世界条件**：光照、天气和视角的极端变化
3. **高分辨率**：2048×2048 像素图像需要大量处理能力
4. **类别不平衡**：某些标志类型比其他类型更常见
5. **密集场景**：单张图像中可能出现多个标志
6. **部分遮挡**：标志可能被车辆、植被或结构部分遮挡

这些挑战使 TT100K 成为开发鲁棒检测算法的宝贵基准。

### 如何处理 TT100K 中的大图像尺寸？

TT100K 数据集使用 2048×2048 像素图像，可能资源密集。以下是推荐的策略：

**训练时：**

```python
# 选项 1：调整为标准 YOLO 尺寸
model.train(data="TT100K.yaml", imgsz=640, batch=16)

# 选项 2：使用更大尺寸以获得更好的小物体检测
model.train(data="TT100K.yaml", imgsz=1280, batch=4)

# 选项 3：多尺度训练
model.train(data="TT100K.yaml", imgsz=640, scale=0.5)  # 以不同尺度训练
```

**建议：**

- 初始实验从 `imgsz=640` 开始
- 如果有足够的 GPU 内存（24GB+），使用 `imgsz=1280`
- 对于非常小的标志，考虑分块策略
- 使用梯度累积来模拟更大的批量大小
