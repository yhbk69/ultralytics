# 持续集成 (CI)

持续集成 (CI) 是软件开发的核心环节，它涉及将更改合并并进行自动化测试。CI 使我们能够在开发过程中尽早、频繁地发现问题，从而保持高质量的代码。在 Ultralytics，我们使用各种 CI 测试来确保代码库的质量和完整性。

## CI 操作

以下是我们的 CI 操作简要说明：

- **[CI](https://github.com/ultralytics/ultralytics/actions/workflows/ci.yml)：** 这是我们主要的 CI 测试，涉及运行单元测试、代码风格检查，有时还会根据仓库情况运行更全面的测试。
- **[Docker 部署](https://github.com/ultralytics/ultralytics/actions/workflows/docker.yml)：** 此测试使用 Docker 检查项目的部署情况，确保 Dockerfile 及相关脚本正常工作。
- **[死链检查](https://github.com/ultralytics/ultralytics/actions/workflows/links.yml)：** 此测试扫描代码库中 Markdown 或 HTML 文件里是否存在损坏或失效的链接。
- **[CodeQL](https://github.com/ultralytics/ultralytics/actions/workflows/codeql.yaml)：** CodeQL 是 GitHub 提供的一款工具，可对代码进行语义分析，帮助发现潜在的安全漏洞并维护高质量代码。
- **[PyPI 发布](https://github.com/ultralytics/ultralytics/actions/workflows/publish.yml)：** 此测试检查项目是否能够无错误地打包并发布到 PyPI。

## 代码覆盖率

代码覆盖率是一个指标，表示在运行测试时代码库中被执行的百分比。它能够揭示测试对代码的覆盖程度，对于识别应用中未测试的部分至关重要。高代码覆盖率通常意味着更低的缺陷概率。然而，必须理解的是，代码覆盖率并不能保证没有缺陷，它仅表明代码的哪些部分已被测试执行。

### 与 [codecov.io](https://about.codecov.io/) 的集成

在 Ultralytics，我们已将仓库与 [codecov.io](https://about.codecov.io/) 集成，这是一个用于测量和可视化代码覆盖率的流行在线平台。Codecov 提供详细的洞察、提交之间的覆盖率比较，以及直接在代码上的可视化覆盖层，标明哪些行已被覆盖。

通过与 Codecov 集成，我们旨在通过关注可能容易出错或需要进一步测试的区域，来维护和提高代码质量。

## 常见问题解答

### 什么是 Ultralytics 中的持续集成 (CI)？

Ultralytics 中的持续集成 (CI) 涉及自动集成和测试代码更改，以确保高标准的质量。我们的 CI 设置包括运行[单元测试、代码风格检查和全面测试](https://github.com/ultralytics/ultralytics/actions/workflows/ci.yml)。此外，我们还执行 [Docker 部署](https://github.com/ultralytics/ultralytics/actions/workflows/docker.yml)、[死链检查](https://github.com/ultralytics/ultralytics/actions/workflows/links.yml)、用于发现安全漏洞的 [CodeQL 分析](https://github.com/ultralytics/ultralytics/actions/workflows/codeql.yaml) 以及用于打包和分发软件的 [PyPI 发布](https://github.com/ultralytics/ultralytics/actions/workflows/publish.yml)。

### Ultralytics 如何检查文档和代码中的死链？

Ultralytics 使用专门的 CI 操作来[检查死链](https://github.com/ultralytics/ultralytics/actions/workflows/links.yml)，扫描 Markdown 和 HTML 文件中的损坏或失效链接。这有助于通过识别死链来维护文档完整性，确保用户始终能够访问准确且有效的资源。

### 为什么 CodeQL 分析对 Ultralytics 的代码库很重要？

[CodeQL 分析](https://github.com/ultralytics/ultralytics/actions/workflows/codeql.yaml) 对 Ultralytics 至关重要，因为它执行语义代码分析以发现潜在的安全漏洞并维护高质量标准。通过 CodeQL，我们可以主动识别和降低代码中的风险，帮助交付健壮且安全的[软件解决方案](https://www.ultralytics.com/solutions)。

### Ultralytics 如何利用 Docker 进行部署？

Ultralytics 通过专门的 CI 操作使用 Docker 来验证项目的部署。此过程确保我们的 [Dockerfile 及相关脚本](https://github.com/ultralytics/ultralytics/actions/workflows/docker.yml) 正常工作，从而为可扩展且可靠的 AI 解决方案提供一致且可重现的部署环境。

### 自动化 PyPI 发布在 Ultralytics 中扮演什么角色？

自动化 [PyPI 发布](https://github.com/ultralytics/ultralytics/actions/workflows/publish.yml) 确保我们的项目能够无错误地打包并发布。此步骤对于分发 Ultralytics 的 Python 包至关重要，使用户能够通过 [Python Package Index (PyPI)](https://pypi.org/project/ultralytics/) 轻松安装和使用我们的工具。

### Ultralytics 如何衡量代码覆盖率，为什么它很重要？

Ultralytics 通过与 [Codecov](https://app.codecov.io/github/ultralytics/ultralytics) 集成来衡量代码覆盖率，从而了解测试期间代码库被执行的覆盖程度。高代码覆盖率可以表明代码经过了充分测试，有助于发现可能容易出错但未测试的区域。详细的代码覆盖率指标可以通过主要仓库上显示的徽章或直接在 [Codecov](https://app.codecov.io/gh/ultralytics/ultralytics) 上查看。
