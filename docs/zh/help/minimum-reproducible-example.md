# 为 Bug 报告创建最小可复现示例

在向 [Ultralytics](https://www.ultralytics.com/) [YOLO](https://github.com/ultralytics) 仓库提交 Bug 报告时，提供一个[最小可复现示例（MRE）](https://stackoverflow.com/help/minimal-reproducible-example)至关重要。MRE 是一段小巧、自包含的代码，用于展示你遇到的问题。提供 MRE 有助于维护者和贡献者理解问题并更高效地进行修复。本指南将说明在向 Ultralytics YOLO 仓库提交 Bug 报告时如何创建 MRE。

## 1. 隔离问题

创建 MRE 的第一步是隔离问题。移除与问题无直接关联的任何不必要的代码或依赖项。聚焦于导致问题的特定代码部分，并剔除所有无关的片段。

## 2. 使用公开模型和数据集

在创建 MRE 时，使用公开可用的模型和数据集来复现问题。例如，使用 `yolo26n.pt` 模型和 `coco8.yaml` 数据集。这样可以确保维护者和贡献者能够轻松运行你的示例并调查问题，而无需访问专有数据或自定义模型。

## 3. 包含所有必要的依赖项

确保你的 MRE 中包含所有必要的依赖项。如果你的代码依赖外部库，请注明所需的包及其版本。理想情况下，如果你已安装 `ultralytics`，可以使用 `yolo checks` 列出依赖项；对于其他工具，则可以使用 `pip list`。

## 4. 清晰描述问题

对你遇到的问题提供清晰简洁的描述。说明预期行为和你实际遇到的行为。如果适用，请附上任何相关的错误消息或日志。

## 5. 正确格式化代码

在问题描述中使用代码块正确格式化你的代码。这能让其他人更容易阅读和理解你的代码。在 GitHub 中，你可以用三个反引号（\```）包裹代码并指定语言来创建代码块：

````bash
```python
# 你的 Python 代码放在这里
```
````

## 6. 测试你的 MRE

在提交 MRE 之前，先对其进行测试，确保它能准确复现问题。确保其他人无需任何问题或修改即可运行你的示例。

## MRE 示例

以下是一个假设性 Bug 报告的 MRE 示例：

**Bug 描述：**

在对 0 通道图像进行推理时，我遇到了一个与输入张量维度相关的错误。

**MRE：**

```python
import torch

from ultralytics import YOLO

# 加载模型
model = YOLO("yolo26n.pt")

# 加载一张 0 通道图像
image = torch.rand(1, 0, 640, 640)

# 运行模型
results = model(image)
```

**错误消息：**

```
RuntimeError: Expected input[1, 0, 640, 640] to have 3 channels, but got 0 channels instead
```

**依赖项：**

- `torch==2.3.0`
- `ultralytics==8.2.0`

在此示例中，MRE 用最少的代码展示了问题，使用了公开模型（`"yolo26n.pt"`），包含了所有必要的依赖项，并提供了清晰的问题描述以及错误消息。

遵循这些指南，你将帮助 Ultralytics YOLO 仓库的维护者和[贡献者](https://github.com/ultralytics/ultralytics/graphs/contributors)更高效地理解和解决你的问题。
