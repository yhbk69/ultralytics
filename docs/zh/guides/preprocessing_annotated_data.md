## 数据预处理的重要性

我们已经谨慎地收集和标注了数据，并考虑了多种因素。那么，是什么让数据预处理对计算机视觉项目如此重要呢？数据预处理的核心是将数据转换为适合训练的格式，以减少计算负载并帮助提升模型性能。以下是预处理能够解决的原始数据中常见的几个问题：

- **噪声**：数据中不相关或随机的变化。
- **不一致性**：图像大小、格式和质量的差异。
- **不平衡**：数据集中各类别或分类分布不均衡。

## 数据预处理技术

数据预处理中最首要的步骤之一是调整大小。有些模型被设计为能够处理可变输入尺寸，但许多模型需要一致的输入尺寸。调整图像大小使其统一，并降低计算复杂度。

### 调整图像大小

可以使用以下方法调整图像大小：

- **双线性插值**：通过对最近四个像素值进行加权平均来平滑像素值。
- **最近邻插值**：直接使用最近的像素值而不进行平均，导致图像出现块状效果，但计算速度更快。

为了让调整大小更简便，可以使用以下工具：

- **[OpenCV](https://www.ultralytics.com/glossary/opencv)**：一个流行的计算机视觉库，提供了丰富的图像处理功能。
- **PIL（Pillow）**：一个用于打开、处理和保存图像文件的 Python 图像库。

对于 YOLO26，[模型训练](../modes/train.md)时的 `imgsz` 参数允许灵活的输入尺寸。当设置为特定尺寸（如 640）时，模型会调整输入图像，使其最大维度为 640 像素，同时保持原始宽高比。

通过评估模型和数据集的具体需求，可以判断调整大小是否是必要的预处理步骤，或者模型是否能够高效处理不同尺寸的图像。

### 归一化像素值

另一个预处理技术是归一化。归一化将像素值缩放到标准范围，有助于训练过程中更快收敛并提高模型性能。以下是常见的归一化技术：

- **最小-最大缩放**：将像素值缩放到 0 到 1 的范围。
- **Z-Score 归一化**：基于像素值的均值和标准差进行缩放。

对于 YOLO26，归一化作为模型训练预处理流水线的一部分无缝处理。YOLO26 会自动执行多个预处理步骤，包括转换为 RGB、将像素值缩放到 [0, 1] 范围，以及使用预定义的均值和标准差进行归一化。

### 划分数据集

数据清洗完成后，就可以划分数据集了。将数据划分为训练集、验证集和测试集是为了确保模型能够在未见过的数据上进行评估，从而衡量其泛化性能。常见的划分比例是 70% 用于训练，20% 用于验证，10% 用于测试。有多种工具和库可用于数据划分，如 [scikit-learn](https://scikit-learn.org/stable/modules/generated/sklearn.model_selection.train_test_split.html) 或 [TensorFlow](https://www.ultralytics.com/glossary/tensorflow)。

划分数据集时需要考虑以下几点：

- **保持数据分布**：确保各集合中类别的数据分布保持一致。
- **避免数据泄露**：通常情况下，数据增强应在数据集划分之后进行。数据增强和任何其他预处理只应应用于训练集，以防止验证集或测试集的信息影响模型训练。
- **平衡类别**：对于不平衡的数据集，可以考虑在训练集中对少数类别进行过采样或对多数类别进行欠采样等技术。

### 什么是数据增强？

最常被讨论的数据预处理步骤是数据增强。数据增强通过创建图像的修改版本来人为增加数据集的大小。通过增强数据，可以减少[过拟合](https://www.ultralytics.com/glossary/overfitting)并提高模型的泛化能力。

数据增强的其他好处包括：

- **创建更稳健的数据集**：数据增强可以使模型对输入数据中的变化和失真更具鲁棒性，包括光照、方向和尺度的变化。
- **成本效益高**：数据增强是一种成本效益高的增加[训练数据](https://www.ultralytics.com/glossary/training-data)量的方法，无需收集和标注新数据。
- **更好地利用数据**：通过创建新的变体，最大限度地利用每一个可用的数据点。

#### 数据增强方法

常见的增强技术包括翻转、旋转、缩放和颜色调整。多个库（如 [Albumentations](../integrations/albumentations.md)、Imgaug 和 TensorFlow 的 ImageDataGenerator）可以生成这些增强。

对于 YOLO26，可以通过修改数据集配置文件（.yaml 文件）来[增强自定义数据集](../modes/train.md)。在该文件中，可以添加一个 augmentation 部分，指定增强数据的参数。

[Ultralytics YOLO26 仓库](https://github.com/ultralytics/ultralytics/tree/main)支持广泛的数据增强方式。可以应用各种变换，例如：

- 随机裁剪
- 翻转：图像可以水平或垂直翻转。
- 旋转：图像可以按特定角度旋转。
- 畸变

此外，可以通过特定参数调整这些增强技术的强度，以生成更多种类的数据。

## 预处理案例研究

假设有一个项目旨在开发一个模型，使用 YOLO26 检测和分类交通图像中不同类型的车辆。我们已经收集了交通图像并用边界框和标签进行了标注。

以下是该项目每个预处理步骤的具体操作：

- **调整图像大小**：由于 YOLO26 可以处理灵活的输入尺寸并自动执行调整大小，因此不需要手动调整大小。模型将在训练过程中根据指定的 `imgsz` 参数调整图像尺寸。
- **归一化像素值**：YOLO26 在预处理过程中会自动将像素值归一化到 0 到 1 的范围，因此不需要手动操作。
- **划分数据集**：使用 scikit-learn 等工具将数据集划分为训练集（70%）、验证集（20%）和测试集（10%）。
- **[数据增强](https://www.ultralytics.com/glossary/data-augmentation)**：修改数据集配置文件（.yaml），加入随机裁剪、水平翻转和亮度调整等数据增强技术。

这些步骤确保数据集准备妥当，没有任何潜在问题，并准备好进行探索性数据分析（EDA）。

## 探索性数据分析技术

在对数据集进行预处理和增强之后，下一步是通过探索性数据分析获取洞察。EDA 使用统计技术和可视化工具来理解数据中的模式和分布。你可以识别类别不平衡或异常值等问题，并为进一步的数据预处理或模型训练调整做出明智的决策。

### 统计 EDA 技术

统计技术通常从计算基本指标开始，如均值、中位数、标准差和范围。这些指标可以快速概括图像数据集的属性，如像素强度分布。理解这些基本统计量有助于掌握数据的整体质量和特征，从而及早发现任何异常。

### 可视化 EDA 技术

可视化是图像数据集 EDA 的关键。例如，类别不平衡分析是 EDA 的另一个重要方面。它有助于确定某些类别在数据集中是否代表性不足。使用条形图可视化不同图像类别或分类的分布可以快速揭示任何不平衡现象。类似地，可以使用箱线图等可视化工具来识别异常值，突出像素强度或特征分布中的异常情况。异常值检测可以防止异常数据点扭曲结果。

常用的可视化工具包括：

- **直方图和箱线图**：有助于理解像素值分布和识别异常值。
- **散点图**：有助于探索图像特征或标注之间的关系。
- **热力图**：用于可视化像素强度分布或图像中标注特征的空间分布。

### 使用 Ultralytics Platform 进行 EDA

如需以无代码方式进行 EDA，可以将数据集上传到 [Ultralytics Platform](https://platform.ultralytics.com/)。数据集的 `Charts` 选项卡会自动生成上述可视化内容：划分分布、最多类别的计数、图像宽度/高度直方图以及标注位置和图像尺寸的二维热力图。`Images` 选项卡允许以网格、紧凑或表格视图浏览数据，并叠加标注信息，无需编写任何代码即可轻松发现标注错误的样本或不平衡的类别。

## FAQ

### 数据预处理在计算机视觉项目中的重要性是什么？

数据预处理在计算机视觉项目中至关重要，因为它确保数据干净、一致，并以适合模型训练的最佳格式呈现。通过解决原始数据中的噪声、不一致性和不平衡等问题，调整大小、归一化、增强和数据集划分等预处理步骤有助于减少计算负载并提高模型性能。更多详情请参阅[计算机视觉项目的步骤](../guides/steps-of-a-cv-project.md)。

### 如何使用 Ultralytics YOLO 进行数据增强？

使用 Ultralytics YOLO26 进行数据增强时，需要修改数据集配置文件（.yaml）。在此文件中，可以指定各种增强技术，如随机裁剪、水平翻转和亮度调整。这可以通过[此处](../modes/train.md)说明的训练配置来有效完成。数据增强有助于创建更稳健的数据集、减少[过拟合](https://www.ultralytics.com/glossary/overfitting)并提高模型泛化能力。

### 计算机视觉数据的最佳归一化技术是什么？

归一化将像素值缩放到标准范围，以加快训练过程中的收敛速度并提高性能。常见技术包括：

- **最小-最大缩放**：将像素值缩放到 0 到 1 的范围。
- **Z-Score 归一化**：基于像素值的均值和标准差进行缩放。

对于 YOLO26，归一化自动处理，包括转换为 RGB 和像素值缩放。更多信息请参阅[模型训练部分](../modes/train.md)。

### 如何为训练划分标注数据集？

划分数据集的常见做法是将其划分为 70% 用于训练，20% 用于验证，10% 用于测试。重要的是要保持各类别在这些子集中的数据分布，并通过仅在训练集上进行数据增强来避免数据泄露。使用 scikit-learn 或 [TensorFlow](https://www.ultralytics.com/glossary/tensorflow) 等工具进行高效的数据集划分。请参阅[数据集准备指南](../guides/data-collection-and-annotation.md)。

### YOLO26 能否在不手动调整大小的情况下处理不同尺寸的图像？

可以，Ultralytics YOLO26 可以通过模型训练时的 `imgsz` 参数处理不同尺寸的图像。该参数确保图像被调整，使其最大维度与指定尺寸（例如 640 像素）匹配，同时保持宽高比。有关更灵活的输入处理和自动调整，请查看[模型训练部分](../modes/train.md)。
