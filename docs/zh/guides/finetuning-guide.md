# 如何在自定义数据集上微调 YOLO

[微调](https://www.ultralytics.com/glossary/fine-tuning)通过从已学习的权重开始而非随机初始化，使预训练模型适应识别新的类别。与从头开始训练数百个 epoch 不同，微调利用预训练的 [COCO](../datasets/detect/coco.md) 特征，只需很少的时间即可在自定义数据上收敛。

本指南涵盖在自定义数据集上微调 [YOLO26](../models/yolo26.md)，从基础用法到高级技巧，如[层冻结](#freezing-layers)和[两阶段训练](#two-stage-fine-tuning)。

## 微调 vs 从头训练

预训练模型已经从数百万张图像中学会了通用的视觉特征——边缘检测、纹理识别、形状理解。通过微调进行的[迁移学习](https://www.ultralytics.com/glossary/transfer-learning)会复用这些知识，只需教会模型新类别的样子，这就是它收敛更快、所需数据更少的原因。从头训练则丢弃了所有这些知识，迫使模型从像素级模式开始学习一切，需要消耗大量资源。

|                       | 微调                                         | 从头训练                                                     |
| --------------------- | -------------------------------------------- | ------------------------------------------------------------ |
| **初始权重**          | 在 COCO（80 类）上预训练                     | 随机初始化                                                   |
| **命令**              | `YOLO("yolo26n.pt")`                         | `YOLO("yolo26n.yaml")`                                       |
| **收敛速度**          | 更快——主干网络已经训练好                     | 更慢——所有层从零开始学习                                     |
| **数据需求**          | 较低——预训练特征弥补了数据不足               | 较高——模型必须仅从数据集中学习所有特征                       |
| **适用场景**          | 自然图像的自定义类别                         | 与 COCO 差异很大的领域（医学、卫星、雷达图像）               |

!!! tip "微调无需额外代码"

    当使用 `YOLO("yolo26n.pt")` 加载 `.pt` 文件时，预训练权重已存储在模型中。之后调用 `.train(data="custom.yaml")` 会自动将所有兼容的权重迁移到新的模型架构中，重新初始化任何不匹配的层（例如类别数量不同时的检测头），并开始训练。无需手动加载权重、层操作或自定义迁移学习代码。

### 预训练权重迁移的工作原理

当在一个类别数量不同的数据集上微调预训练模型时（例如从 COCO 的 80 类到 5 个自定义类），Ultralytics 会执行形状感知的权重迁移：

1. **主干网络和颈部完全迁移**——这些层提取通用视觉特征，其形状与类别数量无关。
2. **检测头部分重新初始化**——分类输出层（`cv3`、`one2one_cv3`）的形状与类别数量绑定（80 vs 5），因此无法迁移，会被随机初始化。检测头中的边界框回归层（`cv2`、`one2one_cv2`）具有与类别数量无关的固定形状，因此正常迁移。
3. **绝大多数权重都会迁移**。只有检测头中特定于分类的层会被重新初始化——主干网络、颈部和边界框回归分支保持不变。

对于与预训练模型类别数量相同的数据集（例如在另一个 80 类数据集上微调 COCO 预训练权重），100% 的权重都会迁移，包括检测头。

## 基础微调示例

!!! example

    === "Python"

        ```python
        from ultralytics import YOLO

        model = YOLO("yolo26n.pt")  # 加载预训练模型
        model.train(data="path/to/data.yaml", epochs=50, imgsz=640)
        ```

    === "CLI"

        ```bash
        yolo detect train model=yolo26n.pt data=path/to/data.yaml epochs=50 imgsz=640
        ```

### 选择模型大小

较大的模型具有更强的能力，但需要更新的参数也更多，当训练数据有限时可能增加过拟合风险。一个实用的做法是从较小的模型（YOLO26n 或 YOLO26s）开始，仅在验证指标停滞时才扩展到更大的模型。最佳模型大小取决于任务复杂度、类别数量、数据集多样性以及部署可用的硬件。查看完整的 [YOLO26 模型页面](../models/yolo26.md) 了解可用尺寸和性能基准。

## 优化器和学习率选择

默认的 `optimizer=auto` 设置会根据总训练迭代次数选择优化器和学习率：

- **< 10,000 次迭代**（小数据集或少量 epoch）：AdamW，使用自动计算的较低学习率
- **> 10,000 次迭代**（大数据集）：[MuSGD](../reference/optim/muon.md)（Muon+SGD 混合优化器），lr=0.01

对于大多数微调任务，默认设置在无需手动调整的情况下效果良好。在以下情况下考虑显式设置优化器：

- **训练不稳定**（损失激增或发散）：尝试 `optimizer=AdamW, lr0=0.001` 以获得更稳定的收敛
- **在小数据集上微调大模型**：较低的学习率如 `lr0=0.001` 有助于保留预训练特征

!!! warning "自动优化器会覆盖手动设置的 lr0"

    当 `optimizer=auto` 时，`lr0` 和 `momentum` 的值会被忽略。要手动控制学习率，请显式设置优化器：`optimizer=SGD, lr0=0.005`。

## 冻结层

冻结可防止特定层在训练期间更新。当数据集相对于模型容量较小时，这可以加速训练并减少[过拟合](https://www.ultralytics.com/glossary/overfitting)。

`freeze` 参数接受整数或列表。整数 `freeze=10` 冻结前 10 层（0 到 9，对应 YOLO26 的主干网络）。列表可以包含层索引，如 `freeze=[0, 3, 5]` 用于部分冻结主干网络，或包含模块名称字符串，如 `freeze=["23.cv2"]` 用于对层内的特定分支进行细粒度控制。

!!! example

    === "冻结主干网络"

        ```python
        model.train(data="custom.yaml", epochs=50, freeze=10)
        ```

    === "冻结特定层"

        ```python
        model.train(data="custom.yaml", epochs=50, freeze=[0, 1, 2, 3, 4])
        ```

    === "按模块名称冻结"

        ```python
        # 冻结检测头的边界框回归分支
        model.train(data="custom.yaml", epochs=50, freeze=["23.cv2"])
        ```

合适的冻结深度取决于目标领域与预训练数据的相似程度以及可用的训练数据量：

| 场景                     | 建议                      | 理由                                                         |
| ------------------------ | ------------------------- | ------------------------------------------------------------ |
| 大数据集，相似领域       | `freeze=None`（默认）     | 有足够的数据来适应所有层而不会过拟合                         |
| 小数据集，相似领域       | `freeze=10`               | 保留主干网络特征，减少可训练参数                             |
| 非常小的数据集           | `freeze=23`               | 仅训练检测头，最大程度降低过拟合风险                         |
| 与 COCO 差异很大的领域   | `freeze=None`             | 主干网络特征可能无法很好地迁移，需要重新训练                 |

冻结深度也可以作为超参数——尝试几个值（0、5、10）并比较验证 mAP 是找到特定数据集最佳设置的实用方法。

## 微调的关键超参数

微调通常比从头训练需要更少的超参数调整。最重要的参数有：

- **`epochs`**：微调比从头训练收敛更快。从一个适中的值开始，使用 `patience` 在验证指标停滞时提前停止。
- **`patience`**：默认值 100 是为长时间训练运行设计的。将其降低到 10-20 可以避免在已经收敛的运行上浪费时间。
- **`warmup_epochs`**：默认的预热（3 个 epoch）从零开始逐渐增加学习率，这可以防止大的梯度更新在早期迭代中破坏预训练特征。即使对于微调也建议保持默认值。

完整的训练参数列表请参见[训练配置参考](../usage/cfg.md)。

## 两阶段微调

两阶段微调将训练分为两个阶段。第一阶段冻结主干网络，只训练颈部和头部，使检测层能够适应新类别而不破坏预训练特征。第二阶段解冻所有层，以较低的学习率训练整个模型，使主干网络针对目标领域进行精调。

当目标领域与 COCO 差异很大时（医学图像、航拍图像、显微镜图像），这种方法特别有用，因为主干网络可能需要适应，但一次性训练所有内容会导致不稳定。有关使用回调方法自动解冻，请参见[冻结和解冻主干网络](custom-trainer.md#freezing-and-unfreezing-the-backbone)。

!!! example "两阶段微调"

    ```python
    from ultralytics import YOLO

    # 第一阶段：冻结主干网络，训练头部和颈部
    model = YOLO("yolo26n.pt")
    model.train(data="custom.yaml", epochs=20, freeze=10, name="stage1", exist_ok=True)

    # 第二阶段：解冻所有层，以较低学习率微调
    model = YOLO("runs/detect/stage1/weights/best.pt")
    model.train(data="custom.yaml", epochs=30, lr0=0.001, name="stage2", exist_ok=True)
    ```

## 常见问题

### 模型不产生任何预测

- **训练数据不足**：用极少量样本训练是最常见的原因——模型无法从太少的数据中学习或泛化。在排查其他原因之前，确保每个类别有足够多样化的样本。
- **检查数据集路径**：`data.yaml` 中不正确的路径会静默地产生零标签。在训练前运行 `yolo detect val model=yolo26n.pt data=your_data.yaml` 确认标签加载正确。
- **降低置信度阈值**：如果存在预测但被过滤掉了，在推理时尝试 `conf=0.1`。
- **验证类别数量**：确保 `data.yaml` 中的 `nc` 与标签文件中的实际类别数量匹配。

### 验证 mAP 早期停滞

- **添加更多数据**：微调显著受益于额外的训练数据，尤其是具有不同角度、光照和背景的多样化样本。
- **检查类别平衡**：代表性不足的类别 AP 会很低。使用 `cls_pw` 应用逆频率类别加权（中度不平衡从 `cls_pw=0.25` 开始，严重不平衡增加到 `1.0`）。
- **减少数据增强**：对于非常小的数据集，过度的数据增强可能弊大于利。尝试 `mosaic=0.5` 或 `mosaic=0.0`。
- **提高分辨率**：对于包含小目标的数据集，尝试 `imgsz=1280` 以保留细节。

### 微调后在原始类别上性能下降

这被称为灾难性遗忘——模型在仅在新数据上微调时丢失了先前学到的知识。如果不将原始数据集图像与新数据一起包含进来，遗忘几乎是不可避免的。缓解方法：

- **合并数据集**：在微调时将原始类别的样本与新类别一起包含进来。这是防止遗忘的唯一可靠方法。
- **冻结主干网络和颈部**：冻结主干网络和颈部，仅训练检测头，适用于以非常低的学习率进行的短期微调。
- **减少训练 epoch**：模型在新数据上独占训练的时间越长，遗忘就越严重。

## 常见问答

### 微调 YOLO 需要多少图像？

没有固定的最小数量——结果取决于任务复杂度、类别数量以及领域与 COCO 的相似程度。多样化的图像（不同的光照、角度、背景）比原始数量更重要。从现有的数据开始，如果验证指标不足再扩展。

### 如何在自定义数据集上微调 YOLO26？

加载预训练的 `.pt` 文件并调用 `.train()`，传入自定义 `data.yaml` 的路径。Ultralytics 自动处理[权重迁移](https://www.ultralytics.com/glossary/transfer-learning)、检测头重新初始化和优化器选择。完整的代码示例请参见[基础微调](#basic-fine-tuning-example)部分。

### 为什么我微调后的 YOLO 模型检测不到任何东西？

最常见的原因是 `data.yaml` 中的路径不正确（会静默地产生零标签）、YAML 中的 `nc` 与实际标签文件不匹配，或者置信度阈值过高。完整的排查清单请参见[常见问题](#model-produces-no-predictions)。

### 微调 YOLO 应该冻结哪些层？

取决于数据集大小和领域相似性。对于与 COCO 领域相似的小数据集，冻结主干网络（`freeze=10`）可防止过拟合。对于与 COCO 差异很大的领域，保持所有层不冻结（`freeze=None`）可以让主干网络适应。详细建议请参见[冻结层](#freezing-layers)。

### 在新类别上微调 YOLO 时如何防止灾难性遗忘？

将原始类别的样本与新类别一起包含在训练数据中。如果无法做到，冻结更多层（`freeze=10` 或更高）并使用较低的学习率有助于保留预训练知识。更多细节请参见[微调后在原始类别上性能下降](#performance-degrades-on-original-classes-after-fine-tuning)。
