# 性能指标深入解析

## 引言

性能指标是评估[目标检测](https://www.ultralytics.com/glossary/object-detection)模型[准确率](https://www.ultralytics.com/glossary/accuracy)和效率的关键工具。它们揭示了模型在图像中识别和定位目标的有效程度。此外，还有助于理解模型对假正例和假负例的处理能力。这些洞察对于评估和提升模型性能至关重要。在本指南中，我们将探讨与 YOLO26 相关的各种性能指标、它们的意义以及如何解读它们。

## 目标检测指标

首先，我们来讨论一些不仅对 YOLO26 重要，而且广泛适用于不同目标检测模型的指标。

- **[交并比](https://www.ultralytics.com/glossary/intersection-over-union-iou) (IoU)：** IoU 是量化预测[边界框](https://www.ultralytics.com/glossary/bounding-box)与真实边界框之间重叠程度的度量。它在评估目标定位准确性方面起着基础性作用。

- **平均精度 (AP)：** AP 计算精确率-召回率曲线下的面积，提供一个概括模型精确率和召回率表现的单一数值。

- **[平均精度均值](https://www.ultralytics.com/glossary/mean-average-precision-map) (mAP)：** mAP 扩展了 AP 的概念，计算多个目标类别的平均 AP 值。这对于多类别目标检测场景非常有用，可以全面评估模型的性能。

- **精确率与召回率：** 精确率量化了所有正例预测中真正例的比例，评估模型避免假正例的能力。另一方面，召回率计算所有真实正例中被正确检测的比例，衡量模型检测某一类别所有实例的能力。

- **F1 Score：** F1 Score 是精确率和召回率的调和平均数，在同时考虑假正例和假负例的情况下，对模型性能提供平衡的评估。

## 如何计算 YOLO26 模型的指标

接下来，我们介绍可用于计算上述评估指标的 [YOLO26 验证模式](../modes/val.md)。

使用验证模式非常简单。一旦拥有已训练的模型，就可以调用 model.val() 函数。该函数将处理验证数据集并返回各种性能指标。但这些指标意味着什么？又该如何解读它们？

### 解读输出

让我们分解 model.val() 函数的输出，理解每个输出部分。

#### 按类别的指标

输出的一个部分是按类别细分的性能指标。当你试图了解模型在每个特定类别上的表现时，这种细粒度的信息非常有用，尤其是在包含多种目标类别的数据集中。对于数据集中的每个类别，提供以下信息：

- **类别 (Class)**：表示目标类别的名称，如"person"、"car"或"dog"。

- **图像数 (Images)**：该指标告诉你验证集中包含该目标类别的图像数量。

- **实例数 (Instances)**：提供该类别在验证集所有图像中出现的总次数。

- **Box(P, R, mAP50, mAP50-95)**：此指标提供了模型检测目标性能的洞察：
    - **P (精确率)**：检测目标的准确性，表示有多少检测结果是正确的。

    - **R (召回率)**：模型识别图像中所有目标实例的能力。

    - **mAP50**：在交并比 (IoU) 阈值为 0.50 时计算的平均精度均值。它衡量模型在仅考虑"简单"检测时的准确性。

    - **mAP50-95**：在不同 IoU 阈值（从 0.50 到 0.95）下计算的平均精度均值的平均值。它全面反映了模型在不同检测难度级别下的性能。

#### 速度指标

推理速度与准确率同样关键，尤其是在实时目标检测场景中。此部分详细列出了验证过程各阶段所花费的时间，从预处理到后处理。

#### COCO 指标评估

对于在 COCO 数据集上进行验证的用户，会使用 COCO 评估脚本计算额外的指标。这些指标提供了在不同 IoU 阈值以及不同大小目标下的精确率和召回率的洞察。

#### 可视化输出

model.val() 函数除了生成数值指标外，还会产生可视化输出，可以更直观地理解模型的性能。以下是你可以预期的可视化输出：

- **F1 Score 曲线 (`F1_curve.png`)**：该曲线表示不同阈值下的 [F1 Score](https://www.ultralytics.com/glossary/f1-score)。解读此曲线可以洞察模型在不同阈值下假正例和假负例之间的平衡。

- **精确率-召回率曲线 (`PR_curve.png`)**：这是任何分类问题的重要可视化工具，该曲线展示了在不同阈值下精确率与[召回率](https://www.ultralytics.com/glossary/recall)之间的权衡。处理不平衡类别时尤为重要。

- **精确率曲线 (`P_curve.png`)**：不同阈值下精确率值的图形表示。此曲线有助于理解精确率如何随阈值变化。

- **召回率曲线 (`R_curve.png`)**：相应地，此图展示了召回率值在不同阈值下如何变化。

- **[混淆矩阵](https://www.ultralytics.com/glossary/confusion-matrix) (`confusion_matrix.png`)**：混淆矩阵提供了结果的详细视图，展示每个类别的真正例、真负例、假正例和假负例的数量。

- **归一化混淆矩阵 (`confusion_matrix_normalized.png`)**：此可视化是混淆矩阵的归一化版本。它以比例而非原始数量表示数据。这种格式使跨类别的性能比较更加简单。

- **验证批次标签 (`val_batchX_labels.jpg`)**：这些图像展示了验证数据集中不同批次的真实标签。它们根据数据集清晰地显示了目标及其相应位置。

- **验证批次预测 (`val_batchX_pred.jpg`)**：与标签图像相对照，这些可视化显示了 YOLO26 模型对相应批次的预测结果。通过将这些图像与标签图像进行比较，你可以直观地评估模型检测和分类目标的效果。

#### 结果存储

为了便于将来参考，结果会保存到一个目录中，通常命名为 runs/detect/val。

## 选择合适的指标

选择合适的评估指标通常取决于具体的应用场景。

- **mAP：** 适用于对模型性能进行广泛评估。

- **IoU：** 当精确的目标定位至关重要时必不可少。

- **精确率：** 当最大限度地减少误检是优先事项时很重要。

- **召回率：** 当需要检测每个目标实例时至关重要。

- **F1 Score：** 当需要在精确率和召回率之间取得平衡时很有用。

对于实时应用，FPS（每秒帧数）和延迟等速度指标对于确保及时输出结果至关重要。

## 结果解读

理解这些指标很重要。以下是一些常见低分可能表明的情况：

- **低 mAP：** 表明模型可能需要进行全面的优化。

- **低 IoU：** 模型可能难以准确定位目标。尝试不同的边界框方法可能有帮助。

- **低精确率：** 模型可能检测到了太多不存在的目标。调整置信度阈值可能会减少这种情况。

- **低召回率：** 模型可能遗漏了真实目标。改进[特征提取](https://www.ultralytics.com/glossary/feature-extraction)或使用更多数据可能有帮助。

- **不平衡的 F1 Score：** 精确率和召回率之间存在差距。

- **特定类别的 AP 较低：** 这里的低分可以突出显示模型难以处理的类别。

## 案例分析

实际案例可以帮助阐明这些指标在实践中如何运作。

### 案例 1

- **情况：** mAP 和 F1 Score 不理想，但召回率尚可，精确率不佳。

- **解读与行动：** 可能存在过多的错误检测。收紧置信度阈值可以减少这些误检，尽管也可能会略微降低召回率。

### 案例 2

- **情况：** mAP 和召回率可以接受，但 IoU 不足。

- **解读与行动：** 模型检测目标的效果不错，但可能无法精确定位它们。优化边界框预测可能有帮助。

### 案例 3

- **情况：** 尽管整体 mAP 尚可，某些类别的 AP 远低于其他类别。

- **解读与行动：** 这些类别对模型来说可能更具挑战性。为这些类别使用更多数据或在训练期间调整类别权重可能是有益的。

## 常见问题

### 平均精度均值 (mAP) 在评估 YOLO26 模型性能中的意义是什么？

平均精度均值 (mAP) 对于评估 YOLO26 模型至关重要，因为它提供了一个涵盖多个类别精确率和召回率的单一指标。mAP@0.50 在 IoU 阈值为 0.50 时衡量精确率，专注于模型正确检测目标的能力。mAP@0.50:0.95 在一系列 IoU 阈值范围内平均精确率，提供了检测性能的全面评估。高 mAP 分数表明模型有效地平衡了精确率和召回率，这对于[自动驾驶](https://www.ultralytics.com/solutions/ai-in-automotive)和监控系统等需要准确检测和最小误报的应用至关重要。

### 如何解读 YOLO26 目标检测中的交并比 (IoU) 值？

交并比 (IoU) 衡量预测边界框与真实边界框之间的重叠程度。IoU 值范围从 0 到 1，较高的值表示更好的定位准确性。IoU 为 1.0 表示完全对齐。通常，0.50 的 IoU 阈值用于在 mAP 等指标中定义真正例。较低的 IoU 值表明模型在精确目标定位方面存在困难，可以通过优化边界框回归或提高[训练数据集](https://docs.ultralytics.com/datasets)中的标注准确性来改善。

### 为什么 F1 Score 对于评估 YOLO26 模型在目标检测中的表现很重要？

F1 Score 对于评估 YOLO26 模型很重要，因为它提供了精确率和召回率的调和平均数，平衡了假正例和假负例。在处理不平衡数据集或仅靠精确率或召回率不够的应用场景中，它特别有价值。高 F1 Score 表明模型在有效检测目标的同时，最大限度地减少了漏检和误报，使其适用于[安防系统](https://docs.ultralytics.com/guides/security-alarm-system)和[医学影像](https://www.ultralytics.com/solutions/ai-in-healthcare)等关键应用。

### 使用 Ultralytics YOLO26 进行实时目标检测的主要优势是什么？

Ultralytics YOLO26 为实时目标检测提供了多项优势：

- **速度和效率**：针对高速推理进行了优化，适用于需要低延迟的应用。
- **高准确率**：先进的算法确保高 mAP 和 IoU 分数，平衡精确率和召回率。
- **灵活性**：支持多种任务，包括目标检测、分割和分类。
- **易用性**：用户友好的界面、丰富的文档，以及与 Ultralytics 平台（[平台快速入门](../platform/quickstart.md)）等工具的无缝集成。

这使得 YOLO26 成为从自动驾驶车辆到[智慧城市解决方案](https://docs.ultralytics.com/guides/queue-management)等各种应用的理想选择。

### YOLO26 的验证指标如何帮助改进模型性能？

YOLO26 的验证指标如精确率、召回率、mAP 和 IoU 通过提供检测不同方面的洞察来帮助诊断和改进模型性能：

- **精确率**：帮助识别和减少假正例。
- **召回率**：确保检测到所有相关目标。
- **mAP**：提供整体性能快照，指导全面改进。
- **IoU**：帮助微调目标定位准确性。

通过分析这些指标，可以针对特定弱点进行改进，例如调整置信度阈值以提高精确率，或收集更多样化的数据以提升召回率。有关这些指标的详细解释以及如何解读它们，请查看[目标检测指标](#目标检测指标)，并考虑实施[超参数调优](https://docs.ultralytics.com/guides/hyperparameter-tuning)来优化你的模型。
